## grtblog-v2

> 本仓库目前正在开发中，等待开发完成后将发布正式版本取代旧版 grtblog（v1）。

为什么有 v2？

旧版 grtblog（v1）在一年多迭代后积累了大量历史包袱：

多服务、多仓库、多插件早期探索导致架构复杂

部署门槛较高，阻碍外部用户、甚至我自己后续维护

Git 历史混入大量二进制资源，仓库体积快速膨胀

设计系统、内容模型与插件机制的边界逐渐模糊

v2 的目标不是简单重写，而是基于 v1 所有经验重新设计一个：

1. 更简单

单体结构、少依赖、少魔法。
核心功能先跑通，复杂能力按阶段演进。

2. 更轻量

默认 SSG，按需 SSR / API。
资产与静态资源彻底解耦，仓库长期保持干净可维护。

3. 更长期

插件系统、联邦能力、智能友链等会进入 roadmap，
但不进入 v2 的初始交付范围。

v2 本质上是：
用工程化方式给自己一个可持续维护的内容平台。
